# Find the angle

The [KUT] is the author's call, but you can lay out the options and stress them so the choice is informed.

## Steps

1. **Read the finding.** Take the draft or the stated result, and the numbers behind it from `outputs/` where they exist.
2. **Generate angles.** Propose three to five distinct readings of what the finding means. Make them genuinely different, not rewordings. One structural, one cautionary, one optimistic, one contrarian, as the finding allows.
3. **Pressure-test each.** For every angle, state what in the data supports it, what would undercut it, and what a sharp critic would say. On the Claude side, run this challenge in a fresh context so it stays ruthless.
4. **Recommend.** Name the angle that survives best and why. Flag any angle the data cannot actually support, so the author does not reach for it.
5. **Hand back.** Present the options and the recommendation. Do not write the [KUT] into the post. That stays the author's line.
