# Plan first

Before any task that touches more than one file, changes an analysis, or would take real time, enter plan mode. Do not start editing.

## The shape

1. Draft the approach. Which files, which scripts, what the finished post or output looks like.
2. Save the plan to `quality_reports/plans/YYYY-MM-DD_slug.md`.
3. Present it and wait for approval.
4. Once approved, work it through end to end. Come back only for a real decision or a genuine ambiguity.
5. Mark the plan done when finished.

A one line fix or a quick lookup skips the plan. Everything else plans first. A plan on disk survives a long session. A plan held only in context does not.
