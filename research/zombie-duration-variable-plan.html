Three blockers that must be resolved in a DB session

The existing zombie pipeline in zombie_topic1_build.py rests on catalog-vintage column mappings (b125, b131, b166–b168) that conflict with the physical-vintage mappings used in the two audited published posts (b110 as revenue, b152–b153 as net result, b086–b087 / b096–b097 as financial debt). The duration analysis must resolve which mapping is correct before computing any zombie flag.
Blocker 1 — Revenue column b125 (catalog: operating income) vs b110 (physical: operating revenue used in the audited debt and margins posts). Both cannot be correct simultaneously. Run: SELECT report_type, db_column, position_label FROM codes_gfi_db_afs_physical WHERE db_column IN ('b110','b125') AND position_label_language='HR' — then cross-tabulate national sums against the ~150–200 bn EUR macro anchor.
Blocker 2 — Interest expense is unverified The published debt post explicitly declined to report ICR because "kamatni rashodi još nisu pouzdano izmjerni." The OECD zombie series in the existing draft rests on b166+b168, which was never audited. HNB FS 22 Box 4 (the Huljak-nod) calls firm-level interest data in FINA "nepouzdani" outright and imputes a benchmark rate instead. The duration study either inherits this unverified denominator or pivots to an interest-free definition for its headline.
Blocker 3 — foreigncontrol scale Research notes assert 0–1 scale (threshold: ≥ 0.5), but zombie_topic1_build.py line 71 uses foreigncontrol > 50. One is wrong. Verify: SELECT MIN(foreigncontrol), MAX(foreigncontrol), AVG(foreigncontrol) FROM db_afs WHERE foreigncontrol IS NOT NULL AND reportyear=2023. The entire zombie_by_ownership_year.csv output is suspect until this is fixed.

P&L variables — resolve vintage conflict first

Operating performance columns

Vintage conflict The catalog vintage (used in zombie_topic1_build.py) and the physical vintage (used in the audited posts) label overlapping AOP positions differently. Do not mix them. Until the DB session resolves the mapping, the column names below are labelled by their established usage, not assumed physical meaning.
b125 / b110 Operating revenue — UNRESOLVED Conflict — must resolve
Catalog vintage: b125 = poslovni prihodi (operating income). Used as revenue in zombie / zagreb builds. Physical vintage: b110 = operating revenue in the audited debt and margins posts, which passed a stock-flow gate (debt/revenue in (0.001, 2.0)) against the ~150–200 bn EUR national anchor.
b110 and b125 cannot both be operating revenue. The physical codebook labels b110 as operating revenue and b125 as something else. This must be resolved before the EBIT numerator is computed. Recommended action: dump both from codes_gfi_db_afs_physical by report_type and compare 2023 national sums to DZS/HNB corporate-sector revenue totals.
b131 / [physical equivalent] Operating expenses — used in EBIT Unaudited · coverage risk
Catalog: poslovni rashodi. EBIT = b125 − b131 (catalog) = EBIT numerator for OECD and HNB definitions.
gfi_blog_topics.md records that b131 is NULL for ~90% of the full-P&L subset — making b125−b131 near-zero for the vast majority of firms and the resulting margin ~88% flat. The existing zombie build silently drops firms where b131 is NULL (EBIT becomes NaN, ICR comparison is False). Audit coverage before relying on the EBIT series; a robust alternative for EBIT is the physical b152−b153 net-result net of tax if that maps to operating earnings.
b166 + b168 Interest expense — OECD ICR denominator Never audited · disputed reliability
Catalog: b166 = kamate i slični rashodi prema poduzetnicima unutar grupe; b168 = kamate i slični rashodi (vanjski). ICR = EBIT / (b166+b168). Required only for the OECD definition.
The published debt post explicitly declined to present ICR because interest data are unreliable. HNB FS 22 Box 4 (Huljak-nod) labels firm-specific interest figures in FINA as "nepouzdani" and uses an imputed 6% benchmark instead. The existing zombie series therefore rests on an unverified denominator. Decision: either audit b166+b168 coverage and validate against HNB aggregate interest flows, or adopt the HNB imputed-rate approach as the headline and treat the OECD/ICR series as a robustness check only.
Current build: fillna(0) on both columns AND requires interest > 0 for ICR < 1 flag — so any firm with NULL interest is structurally excluded from zombie classification by construction. Report this share.
b184 / b185 / b197 / b198 Net profit and net loss of period Used in Zagreb post · partially validated
Required for: HNB filter (net profit > 0 means still profitable → not zombie), Storz ROA criterion. Construction from the Zagreb plan:
net_result = ( COALESCE(NULLIF(b184, 0), NULLIF(b197, 0), 0) - COALESCE(NULLIF(b185, 0), NULLIF(b198, 0), 0) )
b183 (catalog net result) is dead — only ~1% nonzero in 2024. Never use b183 directly. The COALESCE construction above is the sanctioned approach, attributed to vw_db_afs_financial_subject_year logic.
b141 / [physical depreciation] Depreciation & amortisation — for EBITDA Never queried · coverage unknown
Required only for the Storz / ESRB definition: EBITDA = EBIT + depreciation. Catalog b141 = amortizacija.
No committed script has ever selected this column. Coverage is unknown and likely inherits the b131 sparse-filing problem for abbreviated micro-entity forms. Probe in DB session: SELECT reportyear, AVG(b141 IS NOT NULL) AS cov FROM db_afs WHERE b110 > 0 GROUP BY reportyear. If coverage is < 50%, EBITDA-based definitions will lose most small firms — note this as scope limit or restrict to medium/large firms.
Assets, equity, and investment

Balance sheet columns

b063 / b067 Book equity (capital & reserves) Catalog b067 is structurally dead
b067 (catalog) is the naive negative-equity test column. b063 (physical) is the audited equity column from the debt build, confirmed via balance-identity gate. Required for the negative-equity zombie definition (book equity < 0).
b067 < 0 returns ~0% of firms every year. The form floors equity at zero in the catalog-vintage layout — ~94% of balance-filing firms report b067 = 0. USE b063 (physical). Confirm: SELECT reportyear, SUM(b063 < 0)/COUNT(*) FROM db_afs WHERE b061 > 0 GROUP BY reportyear — should return meaningful shares.
b061 / b065 Total assets b061 audited & published
b061 (physical) is the audited total-assets column; balance identity confirmed within 1% in ≥ 95% of covered rows (|b061 − b108| ≤ 1%). Used as ROA denominator (Storz) and universe filter (b061 > 0).
b065 (catalog total assets) is only ~8% populated AND the share rises over time — using it as a universe filter fabricates a fake growth trend. Never use b065. Always use b061.
b002 Fixed assets (dugotrajna imovina) — for Δ net investment Audited & published (debt post)
Net investment = b002(t) − b002(t−1), computed via self-join on subjecttaxnoid, reportyear−1. Required for Storz / ESRB definition criterion (ii). Consistent in both catalog and physical vintages — same label in both.
Coverage ~87–89% among operating firms 2008–2024 (audited). Δb002 is net change, not gross capex (includes disposals, depreciation, revaluation). Cannot stand in for capex. Negative Δb002 is the criterion — do not interpret as disinvestment without qualification.

Definition 1 of 4

Zombie definitions — variable requirements

① OECD / ICR (Adalet McGowan, Andrews & Millot 2017) ICR denominator unaudited
Formula ICR < 1 for three consecutive years AND age ≥ 10. ICR = EBIT / interest_paid. Source confirmed: Economic Policy 33(96), 2018, p. 693 para. 24.
EBIT = b125 − b131 (resolve vintage first) interest = b166 + b168 (audit required) ICR = EBIT / interest (undefined if interest = 0) zombie_t = (ICR_t < 1) AND (ICR_{t-1} < 1) AND (ICR_{t-2} < 1) AND consecutive years AND age >= 10
Columns needed b125 (or physical rev), b131, b166, b168, subjecttaxnoid, reportyear, godina_osnivanja
Persistence 3 consecutive years. Spell start = first year of the qualifying 3-year window (date back 2 years). Recovery = any year where ICR ≥ 1 breaks the chain.
Interest denominator is unverified and flagged as unreliable by HNB. Use as robustness definition only; do not lead with OECD unless the DB session clears b166+b168 coverage. Firms with zero/NULL interest are structurally excluded — report their share.
② Negative equity (Bonfim et al. 2021; Croatian application: Beriša 2023 CNB) Recommended headline — no interest needed
Formula Book equity < 0 in the previous period, i.e. physical b063_{t−1} < 0. No age filter, no interest data required.
zombie_t = (b063_{t-1} < 0) -- physical equity column OR use b061 vs (b061 - b063) to reconstruct if needed -- For spell definition: require b063 < 0 in years t AND t-1 -- (one-year lag per Bonfim et al.; two consecutive years for robustness variant)
Columns needed b063 (physical equity), b061 (physical assets), subjecttaxnoid, reportyear. Self-join on reportyear − 1.
Why this leads No interest data required. Computable from every form vintage 2002–2024. Captures balance-sheet insolvency rather than cash-flow distress. In Croatia, equity can remain negative for years without exit — exactly the zombie-spell mechanics being studied. Beriša (2023, CNB YES seminar) shows this definition yields a distinct population of smaller firms, complementing ICR-based findings.
b063 must be the physical column (from the audited debt build), not b067 (catalog). b067 < 0 returns near-zero by form design. Confirm physical b063 coverage before the spell analysis.
③ HNB imputed-rate / Huljak-nod (HNB Financijska stabilnost 22, Box 4, May 2021) Croatia-tailored · recommended for duration headline
Formula STEP 1 — weak: EBIT < 0.06 × financial_debt for two consecutive years. STEP 2 — zombie: weak AND net_profit < 0 AND age > 3 AND NOT in exit proceedings.
-- Step 1: weak firm EBIT = b125 − b131 (resolve vintage) fin_debt = (b086 + b087) + (b096 + b097) -- physical, audited imputed_interest = 0.06 * fin_debt -- benchmark rate weak_t = (EBIT < imputed_interest) for t AND t-1 -- Step 2: zombie net_profit = COALESCE(NULLIF(b184,0), NULLIF(b197,0), 0) - COALESCE(NULLIF(b185,0), NULLIF(b198,0), 0) zombie_t = weak_t AND (net_profit < 0) AND (age > 3) AND NOT in_exit_proceedings
Why it works here Built to sidestep the exact data problem of b166+b168 unreliability. The 6% imputed rate is the Croatian median interest on creditor liabilities 2019 — consider updating to a time-varying median for the 2002–2024 panel given rate movements. The "exit proceedings" filter requires a merge with the court register (see Exit section).
Result in CNB data End-2019: ~6% of firms as zombies; ~22% as weak; concentrated in transport, agriculture, manufacturing — NOT primarily tourism. This baseline directly enables the COVID counterfactual: was tourism already drifting zombieward before 2020?
Attribution: HNB FS 22 Box 4 is unsigned. Ivan Huljak's personal authorship is plausible (CNB corporate-sector researcher, confirmed mentor of Beriša 2023) but unverified. Do not cite as "Huljak (2021)" in a post — cite as HNB Financijska stabilnost 22, Okvir 4.
④ Storz et al. / ESRB WP 143 (Havemeister & Horn 2023, adopting Storz ECB WP 2104) Depreciation coverage risk
Formula All three simultaneously for 2 consecutive years: (i) ROA < 0; (ii) Δfixed assets < 0; (iii) EBITDA / financial debt < 5%. No age filter — negative net investment is the anti-startup screen.
ROA = net_result / b061 -- physical assets net_inv = b002_t - b002_{t-1} -- audited, published EBITDA = EBIT + b141 -- depreciation coverage unknown fin_debt = (b086+b087) + (b096+b097) -- physical, audited zombie_t = (ROA < 0) AND (net_inv < 0) AND (EBITDA/fin_debt < 0.05) for both t AND t-1
Key advantage Does not require interest expense at all — only EBITDA (which includes depreciation). The 5% threshold is calibrated to the euro-area median interest rate, similar to HNB's 6%. Lowest zombie share of the four definitions by design; identifies the most distressed subpopulation.
EBITDA requires b141 (depreciation), which has never been queried and has unknown coverage. If coverage is low, restrict Storz to firms filing full P&L forms (typically medium and large — NKD sectors with significant fixed assets). Report the restriction explicitly.

Exit & death measurement

Core principle Do not code disappearance from GFI as death. Croatian filing compliance is imperfect; non-filing is a zombie state, not an exit state. Register status + deletion dates are the canonical death variable.

Three-state model for the duration analysis

Healthy
ICR ≥ 1 / equity ≥ 0
⇄
Zombie

Exit

Transitions: Healthy ↔ Zombie (recoverable); Zombie → Exit (absorbing, competing risk); Healthy → Exit (absorbing, direct). Banerjee & Hofmann (BIS WP 882) find ~25% of zombies eventually exit via death and ~60% formally recover — but recovered zombies underperform and face high relapse probability, so the model should distinguish "recovered (at-risk)" from "never-zombie" if sample size allows.

Pre-2009 exit dates Many firms deleted before 2009 have no OIB in the register (OIB was introduced after the register started). Key the exit merge on MBS and map to OIB where present. For 2002–2008 stečaj openings, datum_stecaja is often empty — supplement from e-Oglasna (XLSX export) where coverage begins ~Sept 2015, and from Narodne novine oglasni dio for anything earlier (not reliably machine-searchable pre-2014).

Duration analysis foundations

Zombie spell mechanics

The persistence trap A 3-year consecutive entry rule (OECD) creates a mechanical minimum spell length of 3 years and delays the dated entry by 2 years. Date the spell start at the FIRST year of the qualifying window — not at year 3. Treat the definition's built-in persistence as a measurement property, not as the hazard function itself. The 2-year Storz/HNB rules have a 2-year minimum spell and a 1-year entry lag.

Spell construction (Python window function pattern)

df = df.sort_values(["subjecttaxnoid","reportyear"]) g = df.groupby("subjecttaxnoid", sort=False) # --- OECD 3-year rolling --- df["z_l1"] = g["icr_lt1"].shift(1) df["z_l2"] = g["icr_lt1"].shift(2) df["yr_l1"] = g["reportyear"].shift(1) df["yr_l2"] = g["reportyear"].shift(2) consec = (df.reportyear - df.yr_l1 == 1) & (df.yr_l1 - df.yr_l2 == 1) df["zombie"] = df["icr_lt1"] & (df.z_l1 == True) & (df.z_l2 == True) & consec # Date spell start back to t-2: df["spell_start"] = np.where(df["zombie"], df["reportyear"] - 2, np.nan) # --- Spell ID and duration --- df["z_start"] = g["zombie"].transform( lambda s: s.ne(s.shift()).cumsum() ) # group ID within firm; filter to zombie==True spells # Transition variable (requires register merge) # state: 0 = healthy, 1 = zombie, 2 = exit (absorbing) # event: 0 = censored, 1 = recovery, 2 = exit df["state"] = np.select( [df["exit_confirmed"], df["zombie"], ~df["zombie"]], [2, 1, 0], default=np.nan )

library(survival); library(survminer); library(mstate) # --- Kaplan-Meier zombie spell duration --- km <- survfit( Surv(spell_length, event_recovery) ~ definition, data = zombie_spells # one row per zombie spell per firm ) ggsurvplot(km, risk.table = TRUE, conf.int = TRUE, palette = c("#2E5BA8","#8B1A1A","#3D7055","#8A5E00"), break.time.by = 1, xlab = "Godina trajanja zombi-spelle") # --- Multi-state (healthy / zombie / exit) --- # tmat: transition matrix (3x3, NA for impossible transitions) tmat <- transMat(x = list(c(2,3), c(1,3), c()), names = c("Healthy","Zombie","Exit")) msdata <- msprep(time = c(NA,"t_zombie","t_exit"), status = c(NA,"zombie_event","exit_event"), data = firm_panel, id = "subjecttaxnoid", trans = tmat) fit_ms <- msfit(coxph(Surv(Tstart,Tstop,status) ~ strata(trans), data = msdata, method = "breslow"), trans = tmat) # --- Annual transition probability matrix --- pt <- probtrans(fit_ms, predt = 0) # P(state j at t | state i at t=0)

For the headline LinkedIn statistic — prosječni zombi traje X godina — use median spell length from Kaplan-Meier across all definitions. Report the range across definitions as the robustness bar. The KM curve itself, showing that recovery probability flattens near-zero after year 3–4, is the companion visual to the median.

Reporting the transition matrix The annual transition probability table (Healthy → Zombie, Zombie → Zombie, Zombie → Exit, Zombie → Healthy) is the post's second headline finding. Banerjee & Hofmann (2018) found the "stay zombie" probability rose from 60% in the late 1980s to 85% by 2016 for listed firms in 14 advanced economies. Croatia's probability for the same metric — on a mostly-unlisted, enforcement-weak system — is the number this post will own.
